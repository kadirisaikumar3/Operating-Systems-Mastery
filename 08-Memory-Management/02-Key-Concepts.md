# 02-Key-Concepts
# 🔑 Key Concepts

Before learning advanced memory techniques, understand these important concepts.

---

# Address Binding

Address Binding is the process of mapping program addresses to physical memory addresses.

Types:

- Compile-Time Binding
- Load-Time Binding
- Execution-Time Binding

---

# Logical Address

Generated by the CPU.

Visible to the program.

Also called:

Virtual Address.

---

# Physical Address

Actual address in RAM.

Generated by the Memory Management Unit (MMU).

---

# Memory Management Unit (MMU)

The MMU translates:

Logical Address

↓

Physical Address

Example:

```

CPU

↓

Logical Address

↓

MMU

↓

Physical Address

↓

RAM

```

---

# Dynamic Loading

Program modules are loaded into memory only when needed.

Advantages:

- Saves Memory
- Faster Startup

---

# Dynamic Linking

Libraries are linked during execution instead of compile time.

Example:

Shared DLL files in Windows.

---

# Swapping

Swapping temporarily moves processes between RAM and disk.

Benefits:

- Better RAM utilization
- Supports multitasking

---

# Memory Protection

Protects one process from accessing another process's memory.

Examples:

- Base Register
- Limit Register
- MMU
