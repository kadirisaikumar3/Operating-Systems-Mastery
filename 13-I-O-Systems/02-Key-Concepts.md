# 02-Key-Concepts
# 🔑 Key Concepts

## Device Controller

Hardware component that manages I/O devices.

Responsibilities

- Data Transfer
- Status Reporting
- Device Control

---

## Device Driver

Software that communicates with device controllers.

Examples

- Printer Driver
- GPU Driver
- Network Driver

---

## Programmed I/O

CPU continuously checks device status.

Advantages

- Simple

Disadvantages

- Wastes CPU time

---

## Interrupt-Driven I/O

Device interrupts CPU when service is required.

Advantages

- Better CPU utilization

---

## Direct Memory Access (DMA)

DMA transfers data directly between device and memory.

Benefits

- Faster Transfer
- Reduced CPU Overhead

---

## I/O Performance Metrics

- Throughput
- Response Time
- Latency
- Utilization

---

## Frequently Asked

- Device Drivers
- Interrupts
- DMA
- Programmed I/O

---

## Summary

DMA is generally the most efficient I/O technique for large data transfers.
